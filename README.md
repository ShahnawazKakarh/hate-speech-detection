## 🧭 Overview

Hate speech detection has been studied across three eras of NLP — bag-of-words, distributed embeddings, and contextual transformers — yet most published comparisons mix one of these with idiosyncratic preprocessing on a single dataset. The lay reader is left guessing whether the headline number reflects the model, the data, or the train-time hygiene.

This repository benchmarks one canonical method from each era on **two public datasets** with **identical preprocessing**, **identical splits**, **identical seed**, and **identical metrics**, then runs a **cross-dataset generalization study** to see how each model holds up under distribution shift.

| Method | Era | What it learns | Cost vs. baseline |
|---|---|---|---|
| **TF-IDF + Logistic Regression** | Classical | Sparse n-gram weights | 1× (free) |
| **Paragraph2Vec (Doc2Vec) + LR** | Distributed embeddings (Djuric et al., 2015) | 200-dim joint comment + word vectors via PV-DBOW | ~10× (gensim training) |
| **DistilBERT fine-tuning** | Contextual transformer | Subword-aware contextual representations | ~100× (3 epochs on CPU/MPS) |

> **Why this matters operationally:** moderation queues live or die on **false negatives** in the hate class. A 94% accurate model that wrongly clears 70% of hate posts is worse than an 83% accurate model that flags 75% of them. The headline metric here is **F1 on the hate class**, not raw accuracy — and the cross-dataset story is at least as important as the in-domain one.

## 📈 Results

> ✅ Live results, generated by `scripts/make_results_table.py` from `artifacts/<run>/metrics.json`. Full breakdown + cross-dataset table in [`results/results.md`](results/results.md).

### In-domain test set

Hate as positive class. Seed 42, default 0.5 decision threshold, identical preprocessing across all six runs.

| Model | Dataset | Acc ↑ | P ↑ | R ↑ | F1 (hate) ↑ | F1 (macro) ↑ | AUC ↑ |
|---|---|---|---|---|---|---|---|
| TF-IDF + LR  | Davidson   | 0.904 | 0.315 | 0.559 | 0.403 | 0.676 | 0.857 |
| TF-IDF + LR  | HateXplain | 0.828 | 0.706 | 0.758 | 0.731 | 0.802 | 0.892 |
| Doc2Vec + LR | Davidson   | 0.555 | 0.094 | 0.776 | 0.167 | 0.432 | 0.722 |
| Doc2Vec + LR | HateXplain | 0.700 | 0.509 | 0.769 | 0.613 | 0.684 | 0.795 |
| DistilBERT   | Davidson   | **0.939** | **0.458** | 0.308 | 0.368 | 0.668 | **0.897** |
| DistilBERT   | HateXplain | **0.850** | **0.759** | 0.754 | **0.757** | **0.824** | **0.909** |

**Three different stories on three different axes.** DistilBERT wins HateXplain cleanly across the board. On Davidson (5.8% hate) DistilBERT wins on AUC but loses F1-hate to TF-IDF at the default threshold — a classic class-imbalance artefact, fixable with threshold tuning (next ablation). Doc2Vec, faithful to Djuric et al. (2015), underperforms on Davidson where there are only ~1,150 hate-class training examples — well below the ~50k the original paper had.

### Cross-dataset generalization

Trained on one corpus, evaluated on the other corpus's test split. The number that tells you whether the model would survive deployment on a new platform.

| Model | Train → Eval | F1 (hate) | F1 (macro) | AUC | Δ vs in-domain (macro-F1) |
|---|---|---|---|---|---|
| TF-IDF + LR  | Davidson → HateXplain | 0.484 | 0.587 | 0.658 | −0.215 |
| TF-IDF + LR  | HateXplain → Davidson | 0.269 | 0.607 | 0.689 | −0.069 |
| Doc2Vec + LR | Davidson → HateXplain | 0.474 | 0.426 | 0.570 | −0.257 |
| Doc2Vec + LR | HateXplain → Davidson | 0.139 | 0.454 | 0.594 | −0.230 |
| DistilBERT   | Davidson → HateXplain | 0.370 | 0.587 | 0.689 | −0.237 |
| DistilBERT   | HateXplain → Davidson | 0.259 | **0.613** | **0.812** | −0.055 |

DistilBERT trained on HateXplain transfers best to Davidson (AUC 0.812, the highest cross-dataset AUC observed), suggesting contextual representations are more transferable than n-gram or static-embedding features when the source corpus has enough class signal. Every model degrades — none of these would deploy cleanly on a new platform without re-tuning. That honest finding is the point.

> The point of this work is **not** to chase the highest possible F1. The contribution is the **clean head-to-head on a fixed preprocessing pipeline plus the cross-dataset story**, which the literature has not done together.

## 📊 Datasets

| Dataset | Role | Size | Classes | License | Notes |
|---|---|---|---|---|---|
| [**Davidson 2017**](https://github.com/t-davidson/hate-speech-and-offensive-language) | Train / val / test #1 | 24,783 tweets | 3 (hate / offensive / neither) | Open (research use) | Twitter; heavily imbalanced — 5.77% hate |
| [**HateXplain**](https://github.com/hate-alert/HateXplain) | Train / val / test #2 | 20,148 posts | 3 (hate / offensive / normal) | CC BY 4.0 | Twitter + Gab; ~31% hate; rationale annotations available |

Both datasets are **public**. The `python -m hsd.data.prepare --dataset all` command downloads them automatically. **No private or scraped data.** Stratified splits and the official HateXplain `post_id_divisions.json` are used to guarantee identical splits across every model.

We binarize to *hate vs non-hate* for the headline table and report 3-way macro-F1 in the paper's appendix.

## 🏗️ Pipeline

```
            ┌────────────────────────────┐
            │   Davidson + HateXplain    │
            │   downloaded automatically │
            └─────────────┬──────────────┘
                          ▼
            ┌────────────────────────────┐
            │  Shared preprocessing      │
            │  URL/mention strip · emoji │
            │  · hashtag-word keep ·     │
            │  lowercase · whitespace    │
            └─────────────┬──────────────┘
                          ▼
        ┌─────────────────┴─────────────────┐
        │           Shared splits           │
        │   Davidson 80/10/10 stratified    │
        │   HateXplain official split       │
        └─────────────────┬─────────────────┘
                          ▼
   ┌──────────────┬───────┴───────┬───────────────┐
   ▼              ▼               ▼               ▼
┌────────┐  ┌──────────┐    ┌──────────┐    ┌──────────────┐
│TF-IDF  │  │ Doc2Vec  │    │DistilBERT│    │Cross-dataset │
│ + LR   │  │ + LR     │    │fine-tune │    │  evaluation  │
└───┬────┘  └────┬─────┘    └─────┬────┘    └──────┬───────┘
    │            │                 │                │
    └────────────┴────────┬────────┴────────────────┘
                          ▼
              ┌──────────────────────┐
              │  Unified metrics:    │
              │  Acc · P · R · F1    │
              │  · F1-macro · AUC    │
              │  → results/ + paper/ │
              └──────────────────────┘
```

Single shared preprocessing + single shared metrics function is what makes any performance gap *attributable to the model*, not the cleaning. Implementations live in:

- `src/hsd/data/preprocess.py` — `clean_text` (used by every model)
- `src/hsd/data/prepare.py` — downloads + binarizes + writes parquet splits
- `src/hsd/models/{tfidf,doc2vec,distilbert}.py` — one file per model
- `src/hsd/evaluate.py` — shared metric implementation

## 🚀 Quickstart

### Install

```bash
git clone https://github.com/ShahnawazKakarh/hate-speech-detection.git
cd hate-speech-detection

pyenv local 3.11.1               # any Python >= 3.11 works
python -m venv .venv
source .venv/bin/activate
pip install --upgrade pip
pip install -e ".[dev]"
pre-commit install               # optional but recommended

cp .env.example .env             # then fill in WANDB_API_KEY / HF_TOKEN if desired
```

### One command per (model, dataset)

```bash
# 1. Download + prepare datasets (cached to data/processed/)
python -m hsd.data.prepare --dataset all

# 2. Train (each config is independent, can run in any order)
python -m hsd.train --config configs/tfidf_davidson.yaml         # ~10 sec
python -m hsd.train --config configs/tfidf_hatexplain.yaml       # ~10 sec
python -m hsd.train --config configs/doc2vec_davidson.yaml       # ~30 sec
python -m hsd.train --config configs/doc2vec_hatexplain.yaml     # ~30 sec
python -m hsd.train --config configs/distilbert_davidson.yaml    # ~15 min on M-series CPU/MPS
python -m hsd.train --config configs/distilbert_hatexplain.yaml  # ~13 min on M-series CPU/MPS

# 3. Cross-dataset evaluation (uses already-trained models)
python scripts/cross_dataset_eval.py

# 4. Regenerate results/ + paper tables
python scripts/make_results_table.py
```

### W&B (optional)

Set `WANDB_API_KEY` in `.env`, then flip `wandb: true` and `report_to: wandb` in any YAML config. Runs land under `qapulsebysk/hate-speech-detection`.

## 🔬 Research notes

Full design rationale, prior-art positioning, and v0.1 → v1.0 plan in [`docs/research_notes.md`](docs/research_notes.md).

Headline design decisions:

- **Identical preprocessing across all models.** Any performance gap is the model, not the cleaning.
- **Single seed (42) per run for v0.1.** Mean ± std across seeds is the obvious v1.0 upgrade.
- **Default 0.5 decision threshold.** Un-tuned, for the cleanest comparison. The Davidson + DistilBERT result demonstrates why threshold tuning matters and is the obvious next ablation.
- **Cross-dataset evaluation as a first-class result.** Most hate-speech papers report in-domain F1 and stop. The number that matters for deployment is the cross-dataset one.
- **Negative results stay in.** Doc2Vec on Davidson is poor; we report it. The contribution is the honest comparison.

## ⚠️ Disclaimer

This is **research code**. It is *not* a production moderation system. Outputs must never be used as the sole basis for content removal, account action, or any other consequential decision affecting real users. Hate-speech classifiers are known to inherit biases from their training data — particularly along racial and dialectal lines (see [Sap et al., 2019](https://aclanthology.org/P19-1163/)) — and any deployment requires its own fairness audit, human-in-the-loop review, and per-platform calibration.
